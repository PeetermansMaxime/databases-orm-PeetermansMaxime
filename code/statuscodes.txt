GET    /species      → alle species ophalen    → 200  # standaard dus n vermeldn
GET    /species/1    → één species ophalen     → 200 # standaard dus n vermelden
POST   /species      → nieuwe species aanmaken → 201
PUT    /species/1    → species updaten         → 200
DELETE /species/1    → species verwijderen     → 204
GET    /species/999  → niet gevonden           → 404


2xx - Succes:
200 - OK            → GET, PUT geslaagd
201 - Created       → POST geslaagd, item aangemaakt
204 - No Content    → DELETE geslaagd, niets teruggeven

4xx - Client fout:
400 - Bad Request       → verkeerde data meegestuurd
401 - Unauthorized      → niet ingelogd
403 - Forbidden         → ingelogd maar geen toegang
404 - Not Found         → item bestaat niet
409 - Conflict          → item bestaat al (duplicate)
422 - Unprocessable     → data formaat klopt niet

5xx - Server fout:
500 - Internal Server Error  → bug in je code
503 - Service Unavailable    → server/database niet bereikbaar

Per HTTP methode:
GET    → succes: 200  | niet gevonden: 404  | fout: 500
POST   → succes: 201  | niet gevonden: -    | fout: 422/400
PUT    → succes: 200  | niet gevonden: 404  | fout: 422/400
DELETE → succes: 204  | niet gevonden: 404  | fout: 500




Ja! FastAPI geeft standaard altijd 200 terug voor elk endpoint — ook POST.
Maar de REST conventie zegt:

POST die iets aanmaakt →   hoort 201 Created te zijn

Dus je moet het expliciet vermelden omdat FastAPI het niet automatisch doet voor POST:
python@router.post("/", response_model=Species, status_code=201)  # ← expliciet want afwijking van standaard
